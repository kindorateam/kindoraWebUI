# AGENTS.md — Working Effectively in this Repo

This document is for AI coding agents and contributors working in this project. It encodes the project’s stack, conventions, and safe operating procedures so changes ship quickly and correctly.

## Project Overview
- React 19 + TypeScript (strict) via Vite 7
- Tailwind CSS v4 + HeroUI component library
- Routing with TanStack React Router (file-based, plugin-generated tree)
- Data fetching with TanStack React Query
- State management with Jotai (single shared store for inside/outside React)
- Axios API client with token refresh + redirect handling
- ESLint + Prettier (Tailwind plugin) strictly enforced in CI-like dev flow

## Runbook
- Install deps: `bun install`
- Dev server: `bun run dev`
- Type-check: `bun run typecheck`
- Lint: `bun run lint` | Auto-fix: `bun run lint:fix`
- Format: `bun run format` | Check: `bun run format:check`
- Build: `bun run build`
- Preview: `bun run preview`

Notes
- Bun is the primary runtime/package manager (see `bun.lock`). If using npm/pnpm locally, keep lockfile churn out of scope unless requested.

## File Layout & Boundaries
- Generated files: Do not edit `src/routeTree.gen.ts` directly. It’s generated by the TanStack Router plugin.
- Routes live in `src/routes`. Use file-based routes with `createFileRoute` and nesting under layout routes like `src/routes/_authenticated.tsx` and `src/routes/_auth.tsx`.
- Global state atoms: `src/stores/*`. Use `appStore` from `src/stores/jotaiStore.ts` if you need atoms outside React.
- API/services layer: `src/services/*`. Do not call `axios` directly from components—use the exported `apiClient`.
- UI system: Tailwind utilities (`src/index.css`) + HeroUI components. New global tokens/utilities belong in `src/index.css`.

## Routing Conventions (TanStack)
- File-based routing is configured in `vite.config.ts` via `@tanstack/router-plugin`.
- New routes: create files under `src/routes`, export `Route` using `createFileRoute`.
- Ignore files from routing by prefixing filename with `-` (plugin’s `routeFileIgnorePrefix`).
- Use `beforeLoad` guards for auth decisions; prefer service helpers (see `src/services/token.service.ts`).
- Root suspense/fallback is defined in `src/routes/__root.tsx` and `src/components/PageLoader.tsx`.

## Auth & State
- Tokens and user: use atoms from `src/stores/auth.store.ts`. Do not use `localStorage` directly—`atomWithStorage` handles persistence.
- In non-React code, read/update atoms via `appStore` (`src/stores/jotaiStore.ts`).
- Auth helpers: `src/services/token.service.ts` (get/clear/refresh tokens) and `src/services/redirect.service.ts` (login redirects + return URL).
- Protected sections use the `_authenticated` layout; guest-only sections use `_auth` with redirects.

## Data Fetching & API
- Use React Query for server state. Prefer colocated hooks per feature (e.g., `useQuery`/`useMutation`) and typed results.
- HTTP calls: use `apiClient` from `src/services/api.service.ts`.
  - Interceptors add `Authorization` and auto-refresh on 401 via `refreshAccessToken()`.
  - Use typed generics: `apiClient.get<MyDto>("/path")`.
- Error handling: convert server errors to `ApplicationError` when building domain helpers; log via `logError` from `src/utils/error.ts`.

## UI & Styling
- Tailwind v4 with custom utilities/tokens defined in `src/index.css` and HeroUI plugin in `src/hero.ts`.
- Prefer HeroUI components for primitives; use Tailwind utilities for layout and spacing.
- SVGs can be imported as React components (see `src/vite-env-override.d.ts`). Place shared icons under `src/components/icons/*`.

## TypeScript & Imports
- TS is strict (`tsconfig.json`). Avoid `any`; prefer precise types and generics.
- Path alias: import app code via `@/*` (configured in `tsconfig.json`).
- Import order and sorting are enforced:
  - Groups: builtin/external → internal/parent/sibling/index → type
  - Alphabetical within groups, newlines between groups
  - JSX props and named imports sorted (see `eslint.config.js` perfectionist rules)

## Error Boundaries
- Wrap top-level app in `ErrorBoundary` (`src/components/error/ErrorBoundary.tsx`).
- For component-level isolation, set `isolate` and pass a `fallback` as needed.

## Environment Variables
- Declared in `src/vite-env.d.ts`. Common keys:
  - `VITE_API_URL` or `VITE_API_BASE_URL` (see note below)
  - `VITE_APP_TITLE`
  - `VITE_GOOGLE_CLIENT_ID`
- Access via `import.meta.env.*`. Do not read from `process.env`.

## What To Avoid
- Don’t edit generated files like `src/routeTree.gen.ts`.
- Don’t bypass the services layer to call `axios` directly in UI.
- Don’t touch browser storage directly; use Jotai atoms with storage.
- Don’t add new global CSS frameworks; extend Tailwind tokens/utilities instead.
- Don’t introduce non-deterministic lint/format settings; follow existing ESLint/Prettier config.

## Validation Checklist (before handing off changes)
- Run `bun run typecheck` and `bun run lint` with no errors.
- Ensure imports/props/exports are alphabetically sorted.
- Verify new routes compile and appear in `src/routeTree.gen.ts` after a dev run.
- For API changes, verify token handling paths (401 → refresh → retry) and redirect behavior for failures.
- Keep changes minimal and scoped; respect existing naming and file patterns.

## Common Recipes
- Add a new protected page
  1) Create `src/routes/_authenticated/my-page.tsx` exporting `Route = createFileRoute('/_authenticated/my-page')({ component })`.
  2) Use `beforeLoad` if extra guard logic is required.
  3) Build UI with HeroUI + Tailwind.
- Add a query hook
  1) Add a function in `src/services/*.service.ts` using `apiClient.get<T>()`.
  2) Create a hook near the feature: `useQuery({ queryKey, queryFn })` with proper typing and caching options.
- Add an authenticated API call
  1) Read token using `getCleanToken()` (rarely needed—`apiClient` sets header automatically).
  2) Handle domain errors via `ApplicationError` and log via `logError`.

## Repo Notes / Known Inconsistencies
- README run instructions reference `bun run index.ts` but the app is Vite React. Prefer the Runbook above. Consider updating `README.md`.
- `api.service.ts` reads `VITE_API_BASE_URL` while `src/vite-env.d.ts` declares `VITE_API_URL`. Standardize on one key (recommended: `VITE_API_BASE_URL`).

---

By default, agents should:
- Keep changes surgical, aligned with this guide.
- Prefer creating/updating small, focused files over large refactors.
- Use existing patterns/utilities first; add new ones only if necessary.
- Include brief rationale in PR/hand-off notes and a quick verification plan.
- Apply KISS, YAGNI, DRY principles.
- Prefer functional programming over OOP.
